# Примеры трансформации промтов

Развёрнутые образцы применения методики `clarify-prompt`: сырой ввод пользователя → готовый промт для агента.

## Пример 1: Задача с неявным контекстом

**Вход (сырой текст пользователя):**
> У нас на сайте есть корзина, нужно чтобы при добавлении товара показывалась
> анимация, ну как обычно делают, и ещё чтобы счётчик обновлялся.
> Кнопка где-то в карточке товара, найди сам.

**Выход (промт для агента):**

```
## Задача: Анимация добавления товара в корзину

### Контекст
На сайте есть каталог товаров с карточками и корзина со счётчиком
количества товаров.

### Предварительные шаги
1. Найди в коде компонент карточки товара и кнопку «Добавить в корзину».
2. Найди компонент корзины и его счётчик товаров.
3. Определи, как сейчас работает добавление (через какой state/store/API).

### Что нужно сделать
1. При клике на кнопку «Добавить в корзину» — воспроизвести анимацию:
   миниатюра товара «летит» к иконке корзины.
2. По завершении анимации — обновить счётчик товаров в корзине
   с коротким bounce-эффектом на цифре.
3. Кнопка на время запроса к API должна быть заблокирована
   (disabled + спиннер или текст «Добавляем...»).

### Граничные случаи
- Если запрос к API упал — показать ошибку, счётчик не менять.
- Если товар уже в корзине — увеличить количество, анимацию воспроизвести.
```

## Пример 2: Задача с размытыми требованиями

**Вход:**
> Сделай чтобы в отчёте были графики, ну нормальные чтобы, данные бери из базы,
> там таблица reports вроде или report_data, не помню

**Выход:**

```
## Задача: Графики в отчёте на основе данных из БД

### Предварительные шаги
1. Найди в схеме базы данных таблицы, связанные с отчётами
   (возможные имена: reports, report_data, или похожие).
2. Изучи структуру найденных таблиц: какие поля, типы данных, связи.
3. Определи, какие числовые поля пригодны для визуализации.

### Что нужно сделать
1. На странице отчёта добавить секцию с графиками.
2. Построить графики на основе числовых данных из найденной таблицы.
3. Использовать библиотеку графиков, которая уже есть в проекте
   (найди в зависимостях). Если нет — предложи вариант перед установкой.

### Ожидаемый результат
- Графики отображаются на странице отчёта.
- Тип графика соответствует данным (линейный для динамики,
  столбчатый для сравнений, круговой для долей).
- Графики адаптивны и читаемы на мобильных устройствах.

### Граничные случаи
- Если данных нет — показать сообщение «Нет данных для отображения».
- Если данных слишком много — ограничить выборку последними N записями
  и добавить пагинацию/прокрутку.
```
