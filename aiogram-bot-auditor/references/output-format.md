# Формат вывода аудита

Сначала вердикт, потом находки по убыванию риска, потом правки.

```markdown
## Аудит бота — вердикт

**Готов к проду:** НЕТ / С ОГОВОРКАМИ / ДА
**Стек:** aiogram 3.x · polling · RedisStorage · systemd
**Проверено:** точки входа, 4 роутера, 2 middleware, FSM, рассылка, юнит-файл

Одной строкой: что роняет/тормозит бота или почему он в порядке.

---

## Находки

### CRITICAL — <краткое имя>
- **Где:** `bot/__main__.py` / `handlers/broadcast.py:42`
- **Что произойдёт:** напр. «синхронный psycopg в async-хендлере блокирует event loop —
  бот перестаёт отвечать всем на время запроса».
- **Почему опасно у тебя:** <привязка к polling/рассылке/масштабу/одному инстансу>
- **Как исправить:** <async-драйвер / run_in_executor / троттлинг / один инстанс …>

### HIGH — <…>
### MEDIUM — <…>
### LOW — <…>

---

## Предлагаемые правки

Для каждой принятой находки — конкретный диф/код (обработчик `TelegramRetryAfter`, graceful
shutdown в `dp.shutdown`, перенос логики в `services/`, RedisStorage, и т.п.).

## Чеклист перед деплоем
- [ ] Гарантирован один polling-инстанс
- [ ] Рассылка: троттлинг + обработка TelegramRetryAfter/TelegramForbiddenError
- [ ] Нет блокирующих операций в event loop
- [ ] FSM на RedisStorage (прод)
- [ ] Graceful shutdown закрывает session/redis/пулы
- [ ] Глобальный @dp.errors() обработчик
- [ ] allowed_updates покрывает нужные типы
- [ ] Токен/секреты из env, не в репозитории
- [ ] Идемпотентность для эффектов (платежи и т.п.)
- [ ] Webhook (если есть): secret_token + TLS за nginx
```

## Принципы вывода

- **Вердикт первым** — «можно ли катить», потом детали.
- **Конкретика:** файл, строка, что именно сломается в проде, а не общие слова.
- **Не раздувай риск:** у бота на 50 пользователей без рассылок флуд-контроль — это LOW/MEDIUM,
  а не CRITICAL. Привязывай уровень к масштабу и режиму запуска.
- **Правки — по согласованию:** сначала отчёт, потом переписываем то, что владелец взял в работу.
- **2.x:** если код на aiogram 2.x — это отдельный вердикт «несовместимо», а не набор 3.x-правок.
